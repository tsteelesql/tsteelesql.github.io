Context

LLM applications are easy to tweak‚Äîand surprisingly hard to evaluate.

Small changes to prompts, models, or retrieval pipelines felt like improvements, but there was no consistent way to prove it.

I built a lightweight framework that instruments real LLM workflows to capture concrete signals‚Äîsemantic similarity, retrieval quality, latency, and cost‚Äîwithout forcing a rewrite of existing code.

The system favors pragmatic engineering choices: synchronous execution for clear debugging, lazy initialization to avoid unnecessary overhead, and portable JSON-based storage to keep results inspectable and comparable results.

The result is a repeatable, inspectable way to tune LLM systems based on evidence instead of intuition.
